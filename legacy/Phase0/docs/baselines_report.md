# Phase 0 — Baselines Report

This document is the central comparison table once every baseline has a run
under its belt. It is regenerated by `python -m Phase0.common.report` once that
script lands; until then it is filled in manually from each baseline's
`results/<id>/metrics.json` and `summary.json`.

## How the numbers are produced

Every baseline runs through `Phase0.common.runner.BaselineRunner` which:

1. Loads temporal periods from `Phase0/data/processed/<dataset>/` via
   `Phase0.common.datasets.load_dataset` (self-contained JSONL reader, no
   external dependencies).
2. For each period: pre-training PPL + probe, training via the baseline's
   `train_period`, post-training PPL + probe + combined score, then full BWT
   matrix re-evaluation across all past periods. Per-period RIR is computed
   from the pre/post combined scores (CAPSEL XIII.4).
3. Writes `metrics.json`, `probes_period*.csv`, `training.log`,
   `config.snapshot.json`, and `summary.json` (with CAPSEL ACC / BWT / FWT)
   to `Phase0/results/<baseline_id>/`.

## What we are looking for

- **B1 should lose badly on BWT.** It is the catastrophic-forgetting floor.
- **B2 vs B1** isolates replay. How much of the gap does replay alone close?
- **B3 vs B2** — does regularisation add anything beyond replay?
- **B4 / B5** — does modular routing (prompts / LoRA experts) match or beat
  full fine-tune? B4 updates ~0.1% of params, B5 updates ~1% per period.
- **B6 / B7** — two column-per-period references. B6 is cheaper (one block vs
  a full column); B7 is the lossless upper bound on BWT at linear parameter
  cost.
- **B8 (BlockStack)** — the Paper A variant: frozen block stack + trainable
  current block + pluggable selector (cross_attn / weighted_sum / gated).
  The key question is whether this sits above B6 (single LLaMA-Pro block) on
  combined-score-last-period while matching B7 on BWT at a fraction of the
  parameter cost.

Numbers landing below B1 on combined-score-last-period indicate a bug in the
baseline — we expect a mild-to-moderate win over naive fine-tune from any
reasonable CL mechanism on two or more periods.
